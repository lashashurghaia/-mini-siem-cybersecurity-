# საბაკალავრო პროექტი: ქსელური Mini-SIEM სისტემა მანქანური სწავლების გამოყენებით

ეს პროექტი გავაკეთე კომპიუტერული მეცნიერების საბაკალავრო პროგრამის ფარგლებში. ჩემი მიზანი იყო შემექმნა მარტივი, მაგრამ რეალურად მომუშავე SIEM სისტემის პროტოტიპი, რომელიც ხელოვნური ინტელექტის (კერძოდ Machine Learning-ის) დახმარებით შეძლებდა ქსელში საეჭვო აქტივობებისა და თავდასხმების აღმოჩენას, რასაც მოჰყვებოდა ავტომატური რეაგირება.

## როგორ მუშაობს სისტემა (არქიტექტურა)

პროექტი დავყავი სამ ძირითად ნაწილად, რათა კოდი ყოფილიყო მოწესრიგებული და მარტივად წასაკითხი:

1. მონაცემების მომზადება (generate_data.py): ვინაიდან რეალური ქსელური ტრაფიკის ბაზები ხშირად ძალიან დიდია ან კონფიდენციალურია, დავწერე სკრიპტი, რომელიც თავად აგენერირებს საჭირო მონაცემებს. ის ქმნის როგორც ჩვეულებრივ, უსაფრთხო მომხმარებლების ქცევას, ასევე შეტევების იმიტაციას (მაგალითად, DDoS ან ბრუტფორს სცენარებს).
2. მოდელის გაწვრთნა (train.py): აქ გამოვიყენე Scikit-learn ბიბლიოთეკა და Random Forest-ის ალგორითმი. კოდი კითხულობს გენერირებულ მონაცემებს, სწავლობს მათ მახასიათებლებს და მიღებულ მოდელს ინახავს cyber_model.pkl ფაილში, ხოლო სიზუსტის მაჩვენებლებს აგროვებს metrics.json-ში.
3. ვებ პანელი (app.py): მომხმარებლის ინტერფეისი ავაწყვე Streamlit-ზე. დიზაინი სპეციალურად გავაკეთე მშვიდ, მუქ ტონალობებში, რათა დაცვის ცენტრის (SOC) რეალურ პანელებს გავდეს. 

### საფრთხეების კლასიფიკაცია:
მოდელის მუშაობა დავაფუძნე ალბათობებზე, რაც ნიშნავს, რომ ნებისმიერი პაკეტის შემოწმებისას სისტემა ითვლის შეტევის პროცენტულ შანსს და ტრაფიკს ყოფს სამ კატეგორიად:
* უსაფრთხო (Normal): როცა ალბათობა დაბალია (30%-ზე ნაკლები).
* საეჭვო (Suspicious): როცა ალბათობა საშუალოა (30-დან 70%-მდე). სისტემა ამ დროს უბრალოდ აფრთხილებს ადმინისტრატორს.
* კრიტიკული საფრთხე (Critical): როცა ალბათობა მაღალია (70%-ზე მეტი). ამ დროს ამოქმედდება პრევენციის მექანიზმი და IP მისამართი ავტომატურად იბლოკება.



## რა არის საჭირო პროექტის გასაშვებად

სისტემის ლოკალურად ჩასართავად საჭიროა რამდენიმე ბიბლიოთეკის დაყენება და ფაილების რიგითობის დაცვა.

გახსენით ტერმინალი იმ საქაღალდეში, სადაც ფაილებია და მიყევით ამ ნაბიჯებს:

1. ბიბლიოთეკების ინსტალაცია:
```bash
pip install streamlit pandas numpy scikit-learn joblib


2. ბაზის შექმნა:
```bash
python generate_data.py

3. მოდელის დასწავლება:
```bash
python train.py

4. საიტის ჩართვა:
```bash
streamlit run app.py

მართვის პანელის ფუნქციები
საიტზე გადასვლისას მარცხენა მენიუდან შესაძლებელია ოთხი სხვადასხვა გვერდის გამოყენება:

რეალური დროის მონიტორინგი: აქ ხდება ქსელური ნაკადის იმიტაცია. ღილაკზე დაჭერით სისტემა იწყებს პაკეტების გატარებას, რეალურ დროში ამოწმებს მათ მოდელის დახმარებით და ბლოკავს მავნე IP-ებს.

ლოგების ანალიზი (CSV): ამ გვერდიდან შესაძლებელია გარედან შემოტანილი CSV ფაილის ატვირთვა. მოდელი მთლიან ფაილს ერთიანად გადაამუშავებს და გამოიტანს სტატისტიკურ გრაფებს.

მოდელის ანალიტიკა: აქ გამოვაჩინე მოდელის მუშაობის საერთო სიზუსტე (Accuracy) და შეცდომების მატრიცა, რაც საჭიროა იმის სანახავად, თუ რამდენად სწორად მუშაობს ალგორითმი.

პრევენციის ლოგები: ეს არის ჟურნალი, სადაც ავტომატურად იწერება ყველა ის ინციდენტი და დაბლოკილი IP, რომელიც სისტემამ მუშაობის პროცესში აღმოაჩინა.
